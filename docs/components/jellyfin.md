# Jellyfin

| | |
|---|---|
| **LXC ID** | 102 |
| **IP** | 192.168.1.102 |
| **Internal DNS** | `jellyfin.internal` |
| **Public domain** | jellyfin.welpes.com |
| **Port** | 8096 |
| **Ansible role** | `roles/jellyfin` |
| **Resources** | 2048 MB RAM, 2 vCPU, 8 GB disk |

## Purpose

Media server — streams the movies/TV/music libraries to clients (and to
the household via Jellyseerr's request flow).

## Subcomponents

- **Jellyfin** — installed from the official `repo.jellyfin.org` apt
  repository, runs as a systemd service (`jellyfin.service`).
- **Intel QuickSync (VAAPI) hardware transcoding** — `/dev/dri/renderD128`
  and `/dev/dri/card0` are passed through from the host (configured in
  `1_provision_containers.yml` via `lxc_devices`). `encoding.xml` is
  patched to set `HardwareAccelerationType: vaapi`, enable HEVC
  encode/decode, and add `h264`/`hevc`/`vc1` to hardware decoding codecs.
- **Transcode temp path** redirected to `/data/transcodes` (on the 1.8TB
  media disk) — the container's 8GB rootfs would otherwise fill up during
  live-transcode sessions.

## Storage (bind mounts)

- `mp0`: `/mnt/ssd2tb` -> `/data` — read access to the full media library
  (movies/series/music), plus the `/data/transcodes` scratch directory.
- `mp1`: `/mnt/ssd2tb/configs/jellyfin` -> `/var/lib/jellyfin` — library
  database, user accounts, watch state, plugin data. Survives container
  rebuilds. Backed up nightly (excludes `cache`, `log`, `metadata`,
  `transcodes`).

## Notes

- `/etc/jellyfin/encoding.xml` is rootfs-only (not bind-mounted) and is
  regenerated by Jellyfin on first start of a fresh container — the role
  waits for it to exist before patching it.
- Jellyseerr (LXC 109) reuses Jellyfin's API key (read from
  `/var/lib/jellyseerr/settings.json`) for the on-demand "Jellyfin
  Refresh" task triggered from the Homepage dashboard.

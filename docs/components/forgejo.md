## Purpose

Self-hosted Git server — the source of truth for this Ansible repo
(`homelab-ansible`). Semaphore clones from here (`origin`); a push mirror
keeps a copy on GitHub.

## Subcomponents

- **Forgejo** — single Go binary, downloaded from the latest Codeberg
  release (`forgejo/forgejo`) to `/usr/local/bin/forgejo`.
- Runs as a dedicated `forgejo` system user (`/bin/bash`, home
  `/var/lib/forgejo`), systemd service `forgejo.service`.
- **Config**: `/etc/forgejo/app.ini`. `SECRET_KEY`/`INTERNAL_TOKEN` are
  generated once and read back from the existing `app.ini` on every
  re-run (changing them would invalidate sessions). `JWT_SECRET` /
  `LFS_JWT_SECRET` are auto-generated by Forgejo itself on first start and
  similarly preserved across re-runs.
- On first run, creates an admin user (`forgejo_admin_user` /
  `forgejo_admin_password`, vault) and the `homelab-ansible` repo via the
  API.
- **GitHub push mirror** — `homelab-ansible` is configured (via
  `github_mirror_url`/`github_mirror_username`/`github_mirror_token`,
  vault) to push-mirror to GitHub every 8h or on commit.

## Storage (bind mounts)

- `mp0`: `{{ appdata_base }}/forgejo` (`/opt/appdata/forgejo`) ->
  `/var/lib/forgejo` — repos, database, attachments, LFS data. Backed up
  nightly (excludes `data/sessions`, `data/tmp`, `data/indexers`,
  `data/repo-archive`).

## Notes

- A dedicated SSH key (`semaphore-automation@homeserver`) is authorized
  for Semaphore to clone/pull from this repo and to SSH into every other
  LXC when running playbooks.
